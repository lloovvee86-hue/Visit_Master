# 클라우드 배포 및 상시 가동 가이드 (Always-On)

대표님의 노트북이 꺼져 있어도 방문객이 언제든 QR코드를 통해 시스템을 이용할 수 있도록 웹 폼을 클라우드에 배치하는 방법을 안내합니다.

## 1. 현재 구조의 한계와 해결책
- **현재**: 노트북의 로컬 서버(`localhost`)에서만 열림 → **노트북 종료 시 작동 불가**
- **해결책**: 상시 가동되는 **무료 클라우드 호스팅**에 웹 폼(HTML/CSS/JS)을 업로드

## 2. 추천 배포 방법

### 방법 A: GitHub Pages (가장 권장) - 현재 준비 완료
이미 `Visit_Master` 폴더에 로컬 Git 저장소 설정을 마쳤습니다. 다음 단계만 따라해 주세요:

1. **GitHub** 웹사이트에서 새로운 저장소(Repository)를 만듭니다 (예: `Visit_Master`).
2. 아래 명령어를 터미널에서 순서대로 입력하여 코드를 올립니다:
   ```bash
   git remote add origin https://github.com/대표님아이디/Visit_Master.git
   git branch -M main
   git push -u origin main
   ```
3. GitHub 저장소의 **Settings > Pages** 메뉴로 이동합니다.
4. Build and deployment의 Branch를 **main**으로 설정하고 **Save**를 누릅니다.
5. 잠시 후 `https://대표님아이디.github.io/Visit_Master` 주소로 상시 접속이 가능해집니다.

### 방법 B: Azure Static Web Apps (기업용 추천)
... (기존 내용)
1. 풀무원에서 사용하는 Azure 계정이 있다면 활용 가능합니다.
2. 정적 웹앱 서비스를 생성하고 소스코드를 연결하면 24시간 안정적으로 호스팅됩니다.

### 방법 C: 사내 웹 서버 활용
1. 풀무원 내부에 항상 켜져 있는 웹 서버가 있다면, `Visit_Master` 폴더의 파일을 해당 서버의 웹 경로에 복사만 하면 됩니다.

## 3. 요약
- **파워 오토메이트**: 이미 클라우드(MS)에 있으므로 항상 켜져 있습니다.
- **방문객 웹 폼**: 위의 방법 중 하나로 클라우드에 올리면 노트북 전원과 상관없이 365일 작동합니다.

이후 QR 코드는 노트북 주소(`localhost`)가 아닌, **클라우드 주소**로 생성하여 로비에 비치하시면 됩니다. 🫡
